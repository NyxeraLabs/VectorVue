# VectorVue + SpectraStrike First-Run Guided Demo (Detailed)

## Purpose
This guide provides a granular onboarding workflow for:
1. Pre-seeded guided demonstration mode.
2. Clean first-run assisted mode with zeroed data.

It includes Web UI and TUI operations for both applications.

## Prerequisites
1. Docker + Docker Compose installed.
2. Repositories available:
- `/home/xoce/Workspace/VectorVue`
- `/home/xoce/Workspace/SpectraStrike`
3. Local federation certs generated by project defaults.

## Mode A: Guided Demo With Seeded Multi-Tenant Data

### Step 1. Start local federation
1. In `VectorVue`:
```bash
cd /home/xoce/Workspace/VectorVue
make local-federation-up
```

### Step 2. Seed both platforms
1. In `VectorVue`:
```bash
make demo-seed
```
This action seeds:
- VectorVue database (ACME + Globex tenant datasets, findings, remediation, reports).
- SpectraStrike runtime demo datasets (playbooks and queued executions for both tenant IDs).

### Step 3. Execute guided UI flow
1. Open VectorVue Nexus guided mode:
- `https://127.0.0.1/portal/nexus?demo=true&source=spectrastrike`
2. Open Validation guided mode:
- `https://127.0.0.1/portal/validation?demo=true&source=nexus`
3. Open SpectraStrike workflow guided mode:
- `https://127.0.0.1:18443/ui/dashboard/workflow`

### Step 4. Verify all key panels
1. VectorVue Analytics:
- `https://127.0.0.1/portal/analytics`
2. SpectraStrike Telemetry:
- `https://127.0.0.1:18443/ui/dashboard/telemetry`
3. SpectraStrike ASM:
- `https://127.0.0.1:18443/ui/dashboard/asm`

### Step 5. Guided TUI walkthrough
1. VectorVue TUI:
```bash
cd /home/xoce/Workspace/VectorVue
make run-tui
```
2. SpectraStrike TUI:
```bash
cd /home/xoce/Workspace/SpectraStrike
make open-tui
```

## Mode B: Clean First-Run Assisted Configuration

### Step 1. Reset both sides
1. In `VectorVue`:
```bash
cd /home/xoce/Workspace/VectorVue
make demo-reset
```
This action resets:
- VectorVue PostgreSQL schema.
- SpectraStrike PostgreSQL DB and runtime demo stores.

### Step 2. Start federation again
1. In `VectorVue`:
```bash
make local-federation-up
```

### Step 3. Start assisted first-run from clean baseline
1. Open SpectraStrike guided workflow:
- `https://127.0.0.1:18443/ui/dashboard/workflow`
2. Complete guided steps with empty baseline.
3. Configure your own tenant operations, campaigns, and controls.

### Step 4. Configure through both TUIs
1. VectorVue:
```bash
cd /home/xoce/Workspace/VectorVue
make run-tui
```
2. SpectraStrike:
```bash
cd /home/xoce/Workspace/SpectraStrike
make open-tui
```

## Recommended Training + Handover Sequence
1. `make local-federation-up`
2. `make demo-seed`
3. Full guided flow across SpectraStrike -> Nexus -> VectorVue.
4. `make demo-reset`
5. Clean assisted first-run for real environment setup.

## Notes
- Cross-app routing is HTTPS-only.
- If stale frontend artifacts appear, rebuild UI images:
- VectorVue: `make portal-rebuild-ui`
- SpectraStrike: `make -C ../SpectraStrike ui-web-rebuild`
